# SauceFlow E2E — E-Commerce Automation Hub

Automated end-to-end e-commerce flow using **Flask** (backend) + **Playwright** (browser robot) against the [SauceDemo](https://www.saucedemo.com) store.

The user types a product name and a maximum price. A Playwright robot logs into SauceDemo, collects all products, filters by name and price, displays results in a table, and executes a full checkout. A screenshot of the order confirmation is saved at the end.

---

## Requirements

- Python 3.11+
- [Playwright](https://playwright.dev/python/) Chromium browser

---

## Installation

```bash
cd ecommerce-automation-hub
pip install -r requirements.txt
playwright install chromium
```

Create a `.env` file in the project root:

```env
SAUCE_USERNAME=standard_user
SAUCE_PASSWORD=secret_sauce
CHECKOUT_FIRST_NAME=John
CHECKOUT_LAST_NAME=Doe
CHECKOUT_POSTAL_CODE=12345
```

---

## Running

```bash
python app.py
```

Open `http://127.0.0.1:5002` in your browser.

> Port 5002 is used because 5000 and 5001 are occupied by other projects on the dev machine.

---

## Screen Flow

```
[index.html]        →  user types product name + max price
        ↓
[/loading]          →  animated loading screen (polling every 2s)
        ↓
[/resultados]       →  results table (cheapest product has a badge)
        ↓
[/checkout-status]  →  order confirmation + SauceDemo screenshot
```


```bash

```

**All tests:**


## Technical Notes

- The robot runs in a separate thread; the frontend polls `/status/<id>` every 2s
- Screenshots are saved with an absolute path in `static/screenshots/` and served by Flask
- Logs use structured JSON with `ts`, `level`, `logger`, `requestId`, `step`, `duration_seconds`
- SauceDemo has no native search — the robot collects all products and filters in Python
- Playwright uses `_playwright_registry` to prevent the GC from shutting down the server early
